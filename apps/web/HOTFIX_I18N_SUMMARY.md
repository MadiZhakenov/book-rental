# HOTFIX: Локализация - Упрощенный подход

## Проблема
Next-intl имеет проблемы совместимости с Next.js 16 (Turbopack). Страницы `/ru` и `/kk` возвращают 404.

## Решение
Используем упрощенный подход без middleware:

1. **Удалить middleware.ts** - он вызывает проблемы
2. **Создать отдельные страницы** для каждого языка
3. **Использовать простой Context** для переводов

## Быстрый фикс

### Вариант 1: Статические страницы
Создать:
- `app/ru/page.tsx` - русская версия
- `app/kk/page.tsx` - казахская версия  
- `app/page.tsx` - редирект на `/ru` по умолчанию

### Вариант 2: Вернуться к английскому
Если локализация критична, но не работает:
- Временно вернуть английскую версию
- Добавить переводы позже, когда next-intl обновится для Next.js 16
